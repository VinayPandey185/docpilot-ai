# AI Development Notes

## AI Tools Used

During the development of **DocPilot AI**, I used the following AI tools as development assistants:

- **ChatGPT (GPT-5.5)** – Guidance, debugging, explanations, implementation suggestions, and code reviews.
- **Groq Llama 3.3** – Large Language Model (LLM) used by the application for question answering and agent reasoning.
- **Hugging Face Inference API** – Used to generate vector embeddings for PDF chunks and user queries.

---

# How AI Assisted Development

AI was used as a development assistant throughout the project, similar to consulting official documentation or discussing implementation ideas with a mentor.

Typical use cases included:

- Understanding unfamiliar concepts such as Retrieval-Augmented Generation (RAG) and vector databases.
- Exploring different implementation approaches.
- Debugging errors when I encountered difficult issues.
- Reviewing and improving code quality.
- Refining the user interface and user experience.
- Improving project documentation.

The final implementation, integration, testing, debugging, and architectural decisions were completed, verified, and refined by me.

---

# My Development Process

The project was developed incrementally.

For each feature, I generally followed this workflow:

1. Understand the requirement.
2. Design the implementation.
3. Write or modify the code.
4. Test the feature.
5. Debug any issues.
6. Refine the implementation.

Whenever I encountered unfamiliar concepts or challenging bugs, I used ChatGPT to better understand the problem and explore possible solutions before implementing, testing, and validating them myself.
